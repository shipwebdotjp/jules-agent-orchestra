# jules-agent エージェントガイドライン

このリポジトリで作業するコーディングエージェントのための指針です。

## 優先順位

1.  **CLIとAPIクライアントの安定性**: `jules-agent` CLIおよび `JulesClient` の堅牢性を最優先します。
2.  **ステート管理の正確性**: `.jules-agent/state.json` の読み書きが正確に行われ、不整合が発生しないようにします。
3.  **テストコードの品質**: 機能変更時には必ず対応するテストを更新または追加します。

## 禁止事項

-   **シークレットのハードコード**: APIキー、トークン、その他の機密情報をコードやテストに直接記述しないでください。設定ファイルや環境変数を使用してください。
-   **破壊的なステート変更**: `state.json` のスキーマを壊す変更を、移行手段なしに行わないでください。
-   **アーティファクトの直接編集**: ビルド成果物や生成されたファイル（もしあれば）を直接編集せず、必ずソースファイルを編集してください。

## 作業手順

1.  **現状把握**: `list_files` や `read_file` を使用して、既存のコード構造と `AGENTS.md` を確認してください。
2.  **ソース修正**: コードの修正は常に `src/` ディレクトリ内のファイルに対して行います。
3.  **動作確認**: 修正後は必ず CLI を実行するか、テストを走らせて動作を確認してください。

## テスト方針

-   **フレームワーク**: `pytest` を使用してください。
-   **モック**: HTTPリクエストのモックには `respx` を使用してください。
-   **実行環境**: テストを実行する際は、常に `PYTHONPATH=src` を設定してください（例: `PYTHONPATH=src pytest`）。
-   **カバレッジ**: 重要なロジック（`pipeline.py` や `client.py`）については、正常系だけでなく異常系のテストも記述してください。
